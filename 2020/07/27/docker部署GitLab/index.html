docker部署GitLab

Jul 27, 2020docker

本次测试仅需要Gitlab API，不需要完善的Gitlab服务，因此只做少量设置。

  1. 安装Docker
  2. 部署Gitlab --detach 表示后台运行，不会出现启动过程。去掉之后，会出现启动过程，启动成功也不会停止，而是处于监听状态。
    --publish 使用外部端口映射容器内部端口，注意外部端口不可被占用。例如 --publish 30000:80将容器的80端口映射到外部的30000端口
    --hostname 配置访问地址，只是本地访问的话，设置为localhost，不需要其他的设置。
    --name 用于容器命名，随意，使用docker ps命令查询容器时可以看到容器的名字，是容器的标识。
    --volume 第一个是将容器内部的文件/etc/gitlab 映射到容器外部/srv/gitlab/config。修改容器内部文件时，相应的外部文件也会改变（没有尝试过从外部直接修改会不会影响内部）。设置了这样的映射关系之后，以后启动容器时，映射到这些文件，省去修改的麻烦。
    gitlab/gitlab-ce:latest 注释了gitlab的版本和类型。本地部署分为付费企业版本ee和免费社区版本ce。latest表示是最新的版本，可以替换成想要的版本。

步骤2之后就可以通过访问 http://localhost:80 进行访问了。如果要支持HTTPS协议，需要更进一步的设置。

  1. 配置Gitlab
    测试Gitlab API需要用户信息和access_token，浏览器直接通过Settings -> Access Tokens添加Personal Access Tokens，注意选择有效期和Token的适用范围。生成之后该Tokens只会出现一次，及时存储。

在服务器上进行部署时，只有命令行工具，无法进行这些操作。Gitlab有4种Access Token。其中OAuth2 tokens可以通过HTTP请求获得。
a) 设置用户密码

  • 进入容器，推荐使用以下命令打开一个容器内部的命令行

sudo docker exec -it gitlab /bin/bash

  • 进入gitlab的操作界面

sudo gitlab-rails console production

  • 查找和切换用户，这里查找管理员，一般情况下管理员信息为：{login_name: root, username: admin@example.com}（就是@example.com，没有其他的意思）
  • 修改密码。注意确认密码和保存，save加感叹号
