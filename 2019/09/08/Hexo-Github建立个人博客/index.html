Hexo + Github建立个人博客

Sep 08, 2019Hexo

Hexo加上Github可以快速地建立自己的个人博客，这里记录一下建立个人博客的经验教训。

前期准备

hexo框架及其依赖项安装: nodejs

在hexo installation中有详细的安装步骤：

  • 安装git，配置git。具体配置可以在廖雪峰git教程中找到，主要是密钥ssh的配置。

  • 安装nvm。nvm是nodejs的管理工具，通过nvm可以实现不同版本node的切换。nodejs文件夹的设置影响nvm调用node，很容易失败

  • 安装npm。通过nvm安装的最新版本node是自带npm的。

  1. 初始化个人博客文件夹，在理想的目录下命令行输入以下代码可以初始化一个个人博客文件夹

初始完之后，目录应该至少存在下面几个文件和文件夹

  1. 主题选择

hexo有很多开源的主题可供选择，几乎可以满足任何的审美需求。

  • 首先把选中的主题从GitHub中将开源代码下载下来，放到上面提到的theme文件夹中。主题目录名字应尽量简洁、与其它主题区别，之后需要使用目录名。
  • 打开根config文件，将其中theme值设置为之前的主题名。theme的默认值是landscape，可以看到theme文件夹中有landscape文件夹。
  • 主题目录下也有一个_config.yml，与根config目录相区别，这个yml文件主要用于对主题风格细节的配置。每个主题中的_config.yml大体上一致，也存在区别。

  1. Hexo Generate

    选完主题后，就可以生成博客网站了。在souce/post文件夹下，有一篇示例文章helloword.md，通过generate命令将会在博客网站中生成一篇helloword的文章。
    $ hexo g
    如果报错，说明hexo和生成有关的模块没有安装。通过官网提示和报错信息进行安装即可。

  2. Hexo Server

    博客网站生成之后，可以在本地端口上进行测试，利用命令：
    $ hexo s
    在本地部署网站，默认是端口4000，通过浏览器访问http://localhost:4000查看。

  3. Hexo Deploy

    如果网站没有什么问题，就可以上传到Github Page中，通过互联网让每个人都访问到。Github需要新建一个名字为username.github.io的远程仓库。username需要和个人GitHub账号的名字一致，不然最后个人主页的链接会很繁琐，例如用户名为xiaohu的GitHub用户创建的仓库为xiaoxiaohu.github.io，那么最后个人主页的根地址是https://xiaohu/xiaoxiaohu.github.io，如果名字一致，则个人主页地址为https://xiaohu.github.io，更加简洁。

    在xiaohu.github.io仓库中选择settings，将GitHub Page相关选项设置好即可。

    在本地，根据相关要求，首先需要安装部署相关模块；然后将根config目录中deploy相关键完善一下。然后进行部署，第一次部署需要输入账号，密码。
    $ hexo g

    部署成功之后就可以在<username.github.io>网址中进行访问了。

文章上传

hexo十分方便，做完上面的工作既可以专心写文章了。在上面有提过，通过命令hexo new [layout] <title>新建文章。layout分为三种：

layout meaning
post 在post文件夹中，会直接生成页面
page 在source文件夹里，会直接生成页面，通过相对路径访问到
draft 在draft文件夹中，不会生成页面，通过$ hexo publish draft <title.md>转换成post文件

文章支持markdown格式，十分方面。
